# 开发交付小分队

面向从规格到代码交付的完整开发流程。适合新功能、重构、Bug 修复、测试补齐和工程迁移。

该模板基于以下方法论：

- 先锁定规格，再让 Agent 执行。
- 用 Git、文件系统、任务看板和测试结果承载长期记忆。
- 每次只推进一个清晰的增量任务。
- 通过 LSP / 类型系统 / 测试提供局部上下文和验证闭环。
- Skills 用于固化团队经验，而不是让 Agent 自由发挥。

## Typical Use

1. 用规格负责人澄清需求、非目标和验收标准。
2. 用架构师拆解技术方案、数据结构、模块边界和风险。
3. 用实现工程师按任务增量修改代码。
4. 用测试工程师补齐验证路径。
5. 用审查员做代码、安全和维护性审查。

模板不包含任何密钥、运行状态、任务历史或本地机器路径。
