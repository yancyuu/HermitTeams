# TDD Quality Gate

Use this skill when adding or changing behavior that needs validation.

## Test Strategy

- Cover the happy path.
- Cover boundary cases.
- Cover failure and permission paths.
- Mock external systems when needed.
- Keep tests focused on behavior, not implementation details.

## Validation Report

Every task completion should include:

- commands run
- tests added or updated
- failures encountered
- remaining untested risk

## Guardrails

- Do not mark a task complete just because code was edited.
- Do not claim tests passed unless they were actually run.
- If a test cannot run, explain why and provide the next best validation.
