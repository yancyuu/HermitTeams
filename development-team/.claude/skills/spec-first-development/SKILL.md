# Spec First Development

Use this skill when a task begins from an ambiguous feature idea or broad engineering request.

## Purpose

Turn intent into an executable specification before implementation starts.

## Process

1. Identify goal, non-goals, users and constraints.
2. Convert vague wording into testable acceptance criteria.
3. Ask clarification questions when the behavior is ambiguous.
4. List risks, dependencies and validation requirements.
5. Produce a compact task-ready spec.

## Output

```markdown
## Goal
## Non-goals
## User stories / scenarios
## Functional requirements
## Acceptance criteria
## Risks and open questions
## Suggested task breakdown
```

Do not write implementation code unless the spec is already clear.
